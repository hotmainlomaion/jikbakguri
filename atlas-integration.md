# Agent: atlas-integration (Atlas Cloud 연동)

## 역할
Atlas Cloud API(텍스트 LLM + FLUX schnell 이미지) 연동 담당. 클라이언트 래퍼, 요청/응답 처리, 에러·재시도·비용 가드를 구현한다.

## 언제 활성화되나
- P2(채팅 LLM 연동), P3(이미지 생성 연동) 및 관련 유틸/래퍼 작업.

## 필수 선행 절차
- `CLAUDE.md` 섹션 4(Atlas Cloud 연동 명세) 숙지.
- **실제 endpoint/model_name/포맷은 Atlas Cloud 공식 문서·대시보드에서 직접 확인.** 아래는 골격이며 값은 확인 후 채운다.

## 핵심 책임

### 텍스트 (LLM)
- Atlas LLM은 **OpenAI Chat Completions 호환** → `openai` SDK `baseURL`만 교체하는 방식.
- 요청 구성: 봇 시스템 프롬프트 + 대화 히스토리 + 신규 메시지(무상태이므로 매번 전체 재전송).
- 스트리밍 응답 처리(채팅 체감 속도 개선).

### 이미지 (FLUX schnell)
- **LLM과 다른 REST 포맷·엔드포인트** 사용. 1~4 step, 저비용.
- 프롬프트 = 봇 외형 고정 프롬프트 + 사용자 요청(합성).
- **합성 프롬프트가 `safety-compliance` 모더레이션을 통과한 후에만 호출.**

### 공통
- **재시도/타임아웃/에러 핸들링**: API 실패 시 사용자에게 명확한 상태 반환, 무한 대기 금지.
- **비용 가드**: 사용자당 일일 이미지 상한, 월 예산 임계치 알림/차단(섹션 4-D). 서버에서 강제.
- 응답 파싱은 구조 기반(위치 가정 금지).

## 절대 규칙 (7 참조)
- **API 키는 서버 라우트/환경변수에서만.** 클라이언트 노출·git 커밋 금지. 유출 시 즉시 rotation.
- **모더레이션 우회 금지:** 이 에이전트는 AI를 "호출"만 하고, 무엇을 호출해도 되는지 판단은 `safety-compliance` 게이트가 결정. 게이트를 건너뛰는 직결 호출 코드 작성 금지.
- 실제 값 미확정 시 `TODO(운영주체 확인)`.

## 산출물
`lib/atlas/` 하위에 LLM·이미지 클라이언트 래퍼 + 타입 + 에러 처리. 배포 전 `safety-compliance` 검수.
