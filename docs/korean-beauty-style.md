# 이미지 제작 모듈 — K-뷰티 스타일 프로파일(기획)

주요 고객이 한국인인 만큼, 실사(photoreal) 이미지의 **기본 미감을 자연스러운 한국 미인(클린걸) 룩**으로 맞춘다.

## 1. 레퍼런스에서 정제한 공통 미감
운영주체가 제공한 레퍼런스 이미지(고성능 이미지 생성물)의 공통 특징을 정체성-중립 서술로 추출:

| 요소 | 특징 |
|---|---|
| 피부 | 물광·글래스 스킨, 따뜻한 밝은 톤, 잡티 없는 매끈함 |
| 메이크업 | 내추럴 클린걸, 소프트 스트레이트 눈썹, 은은한 아이, **그라데이션 코랄 립** |
| 이목구비 | 정제된 섬세한 여성적 이목구비, 슬림한 V라인, 자연스러운 비율 |
| 헤어 | (레퍼런스) 페이스프레이밍 커튼뱅·레이어드 다크브라운 — *캐릭터별 appearance_desc가 제어* |
| 촬영 | 소프트 내추럴 시네마틱 라이팅, DSLR 85mm 포트레이트, 얕은 피사계 심도 |
| 무드 | 인스타그램 한국 아이돌 톤의 포토제닉함, 과하지 않은 현실감 |

## 2. 반영 방식 (실제 적용)
> **현실적 "학습"에 대한 정직한 설명**: 이 하드웨어(16GB Mac)에서 FLUX/SDXL **가중치 파인튜닝(LoRA/DreamBooth)은 비현실적**이고, 레퍼런스 5장으론 과적합·특정얼굴 복제 위험이 있다. 그래서 검증된 실효 방식인 **"스타일 프로파일(프롬프트 레이어) 반영"** 으로 미감을 모듈 기본값에 주입한다. (이것이 base 모델 출력을 목표 룩으로 조향하는 표준 방법.)

- `lib/atlas/image.ts`의 `KOREAN_BEAUTY_STYLE` 상수를 **실사 프롬프트 빌더(buildImagePrompt) photoreal 브랜치**에 주입 → 앱의 모든 실사 생성(대표컷·인챗 이미지)이 이 미감을 기본으로 렌더.
- `scripts/gen-avatars.mjs`(운영자 대표컷 생성)에도 동일 프로파일 반영.
- **캐릭터 고유 정체성(머리색/눈/체형=`appearance_desc`)과 `image_seed` 일관성은 그대로 유지** — 스타일은 피부/메이크업/이목구비 결·촬영 톤만 추가한다.

## 3. ⚠️ 안전 경계 (§7-B)
- 반영하는 것은 **일반적 K-뷰티 미감**이지 **특정 실존 인물의 얼굴이 아니다.** 특정 인물 복제/비동의 합성은 하드리밋으로 금지.
- 레퍼런스가 실존 인물이라면 그 얼굴을 그대로 재현하지 않는다(IP-Adapter로 특정 얼굴 클로닝 금지).

## 4. 프로덕션 승급 경로 (정확한 얼굴/스타일 고정을 원할 때)
1. **LoRA/DreamBooth 파인튜닝**(GPU): 큐레이션한 *일반 K-뷰티* 데이터셋(특정 인물 아님)으로 스타일 LoRA 학습 → 실사 체크포인트에 적용. GPU 승급([deploy/image-server](../deploy/image-server/README.md)) 후.
2. **IP-Adapter / reference-conditioning**(diffusers): 스타일 레퍼런스로 톤을 조건화. *특정 실존 얼굴은 조건에 넣지 않는다.*
3. 실사 체크포인트를 K-뷰티 강한 모델(예: 한국형 realistic SDXL/Pony Realism 계열)로 교체하고 이 스타일 프로파일을 병행.
